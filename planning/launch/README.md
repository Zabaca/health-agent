# Launch — App Store + Play Store submission

Docs for getting **Veladon** (`com.zabaca.veladon`, by Zabaca — rebranding from "HealthAgent") submitted to the Apple App Store and Google Play. Build the HTML view with `bun build.ts` from this folder.

## Chapters

1. [App Store + Google Play submission plan](./01-app-store-submission.md) — external requirements (Track A), engineering plan, OAuth portal steps, health-app review specifics, timeline
2. [Vendor compliance alternatives](./02-vendor-compliance-alternates.md) — strategies for when our stack isn't all HIPAA-tier (Strategy A: risk-accept + disclose; B: hybrid storage migration; C: full HIPAA stack; D: encrypt-first)

## Reading order

If short on time: skim **External requirements — start these in parallel with engineering** at the top of chapter 1, then jump to **Critical path**. Read chapter 2 once vendor BAA review (E5) comes back with the actual vendor stances — its strategy choice feeds back into chapter 1's PP brief.
